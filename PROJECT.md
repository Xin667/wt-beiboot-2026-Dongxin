# Body as Input
## Beiboot-Projekt · Webtechnologien · SoSe 2026

**Was, wenn der Browser den Körper versteht?**

Bisher kommunizieren wir mit dem Web fast ausschließlich über Tastatur, Maus und Touch. Dabei ist unser Körper ständig präsent, er bewegt sich, gestikuliert, nähert sich an oder tritt zurück. Das Web ignoriert das meist vollständig.

In diesem Beiboot-Projekt bauen wir eine browserbasierte Library, die Körperdaten aus der Kamera als native Web Events exposed. Hände, Gesten, Pose, Nähe – alles wird zu Ereignissen, auf die jede Web-Anwendung mit einem simplen Event Listener reagieren kann.

## Ziel

Eine wiederverwendbare, dokumentierte JavaScript-Library, die Körperdaten via ML im Browser erkennt und als Custom Browser Events bereitstellt – offen, lizenziert, von anderen Teams weiterbaubar.

## Warum das interessant ist

- Körper als Eingabe ist kein neues Thema – aber eine saubere, entwicklerfreundliche Web-Abstraktion dafür fehlt
- Verschiedene ML-Libraries (MediaPipe, TensorFlow.js u.a.) liefern Rohdaten; der Schritt zur nutzbaren API ist die eigentliche Designaufgabe
- Das Projekt hat einen echten Use-Case-Raum: Präsentationssteuerung, Accessibility, generative Interfaces, Browser-Games
